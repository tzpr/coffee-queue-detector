
redis (message broker between python script and nodejs script)
sudo docker run -d -p 6060:6379 --name coffee-queue redis
sudo docker start coffee-queue

python script (capture images and do the inference)
sudo python3 -m venv venv
source venv/bin/activate
sudo python3 -m pip install -r requirements.txt
python3 queue-inference-jetson-client.py

nodejs script (slack integration api)
touch .env
CLIENT_ID=id_of_the_slack_channel_app
CLIENT_SECRET=secret_of_the_slack_channel_app
SLACK_INTEGRATION_API_PORT=4390
REDIS_PORT=6060
REDIS_HOST=localhost
npm install
node app.js

ngrok proxy (expose jetson nano (localhost) to internet)
./ngrok http 4390

slack (host a slack channel app to query the coffee queue state)
create channel
create app and configure a slash-command to send a request to the slack integration api (via the URL generated by ngrok)
and receive the coffee queue status back.
https://api.slack.com/apps
https://api.slack.com/interactivity/slash-commands
